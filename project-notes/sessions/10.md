**Goal:** Prices tick live on home, category pages, and detail page. The detail page's percentage text and probability bar stay in sync. Flash animation is visible. Reconnects cleanly and resumes subscriptions automatically.

Automated verification on 2026-04-19: `pnpm test -- --run`, `pnpm lint`, and `pnpm build` all passed. Browser QA was then run against the production `pnpm start` build in local Chrome via Playwright-driven automation.

- [x] `features/realtime/subscriptions.ts`:
  - [x] Ref-counted `Map<tokenId, count>`
  - [x] Key ref counts by unique token ID so repeated `PriceCell` mounts for the same token do not duplicate socket subscriptions
  - [x] `subscribe(tokenIds)` increments counts, de-dupes input, diffs the active unique-token set, and sends WS updates only when the effective subscription set changes; use the documented dynamic subscribe behavior
  - [x] `unsubscribe(tokenIds)` decrements, removes when count hits `0`, and only sends the documented unsubscribe message for tokens that truly left the active set
  - [x] Debounce WS sends to 50ms to coalesce mount storms
  - [x] On socket open or reopen, immediately replay the current active token set and verify the transport heartbeat path stays active so subscriptions recover after Wi-Fi drops without a page refresh
  - [x] Start with the simple per-`PriceCell` lifecycle subscription model by default; only add viewport gating if profiler evidence on the large detail-page stress case shows the simple strategy exceeds budget
  - [x] Viewport gating was intentionally not added in this pass; if profiler evidence later proves it necessary, gate at the row/card level rather than per-cell `IntersectionObserver` wiring
- [x] `PriceCell` `useEffect`: `subscribe([tokenId])` on mount, `unsubscribe([tokenId])` on unmount
- [x] Detail-page live parity:
  - [x] The detail `ProbabilityBar` reads from the same live yes-token state as the headline percentage so the bar and text move together
  - [x] Keep the live subscription boundary granular: do not make the whole `MarketRow` rerender on each tick just to animate the bar
