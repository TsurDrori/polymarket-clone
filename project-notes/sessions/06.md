Realignment note (2026-04-19): Session `08R` kept the core Session `06` contracts intact — route-level server fetch, `isEventVisible`, inline empty state, route-level loading/error files, and a server-rendered `EventGrid`. However, the accepted homepage target is now richer than a bare card grid: the right end state is a live-like composition with a hero, breaking-news module, hot-topics module, and an explore-all feed. The currently landed rogue homepage is not accepted as-is; Session `08A` owns that refactor.

- [x] `shared/lib/tags.ts` — add `isEventVisible(event)` returning `event.tags.every(t => !t.forceHide && t.slug !== 'hide-from-new')`. The existing `hasTagSlug` ignores `forceHide`, so a bare slug filter would miss `forceHide`-flagged tags. One helper, reused by every category page.
- [x] `app/page.tsx` (RSC): `await listEvents({ limit: 30, order: 'volume_24hr', ascending: false })` — **snake_case** `volume_24hr` is the Gamma sort token; the camelCase `volume24hr` is the response field name and is silently ignored as a sort param, which would ship an unsorted feed. Filter with `isEventVisible`. No per-event parsing — `parseEventList` already drops invalid events. Render `<><Hydrator events={visible}/><EventGrid events={visible}/></>` — `<Hydrator>` returns `null`, so it's a sibling, not a wrapper.
- [x] `features/events/components/EventGrid.tsx` (+ `.module.css`) — **server component** (pure layout, keeps JS bundle lean; `EventCard` children are the client boundary). `.grid { display: grid; grid-template-columns: minmax(0, 1fr); gap: 16px; } @media (min-width: 768px) { .grid { grid-template-columns: repeat(2, minmax(0, 1fr)); } } @media (min-width: 1024px) { .grid { grid-template-columns: repeat(3, minmax(0, 1fr)); } } @media (min-width: 1200px) { .grid { grid-template-columns: repeat(4, minmax(0, 1fr)); } }` — `minmax(0, 1fr)` prevents grid blow-out from long unbroken title tokens.
- [x] `app/loading.tsx` — grid of 12 `<CardSkeleton>` matching final layout
- [x] `app/error.tsx` — **must start with `'use client'`**. Props: `{ error, unstable_retry }`. Centered message + "Try again" button wired to `unstable_retry()`. In current Next 16 docs, `unstable_retry()` is the preferred recovery API for refetching and rerendering the segment; `reset()` remains available for the narrower "clear error without refetch" case.
- [x] Empty state (`visible.length === 0`): centered message rendered inline from `page.tsx`
- [x] Visual QA at 1280×800, 1024×768, 600×960, 375×812
- [x] **Commit:** `feat(home): render responsive events grid with loading and error states`

Follow-up fixes applied after QA:
- The home grid now stays single-column until `768px`, avoiding the cramped `600px` 2-column tablet layout that showed up in responsive QA.
- `isEventVisible()` now hides only events carrying the explicit `hide-from-new` slug; `forceHide` remains a tag-level visibility concern for `getVisibleTags()`, not a whole-event suppression rule.
